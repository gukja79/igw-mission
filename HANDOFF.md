# igw-mission 인수인계 문서

선교 해외 아웃리치 **오프라인 회계 PWA**. 각 팀 회계가 휴대폰에서 오프라인으로 지출/수입을 입력하면, 연결될 때 팀별 구글시트로 동기화되고 영수증 사진은 구글 드라이브에 저장된다.

> 최종 업데이트 기준 상태: **Phase 2(양방향 동기화) + 잔손질(§12) + Phase 3(사진 관리, §13)까지 완료·검증.** 몽골팀 운영 중(나머지 4팀 미연결). 앱 캐시 버전 `v19`, 최종 커밋 `4e1d037`(HANDOFF는 그 이후).
> **[완료] 환율/환전 자동화(§14) — 시트 + 서버 + 앱 1단계.** 회계가 환전 거래만 입력하면 통화별 가중평균 환율이 자동 산출돼 메인 `I3/6/9`로 흐름. 서버(Code.gs) fx 라우팅 6종 + 코드→이름 변환 배포·검증 완료. 앱(index.html) 환율 직접입력 화면 제거 → 환전 입력/목록/동기화 + 현재환율 카드 + 현지화2(경유통화) 설정🔒, 몽골 1~5번 검증 완료. **남은 §14 작업 = 2단계(환전 수정 + 환전 사진 추가/삭제 관리)** — §14 하단 참고.
> **팀 5개로 확정**: 몽골·중국1·인도네시아1·말레이시아1·스리랑카. 현지화1 통화명 팀 고정(LOCAL1): 몽골 MNT·중국1 CNY·인니 IDR·말레이 MYR·스리랑카 LKR.
> 그 외 남은 일은 운영자 손작업(테스트 데이터 정리, 4팀 환전내역 J3 통화코드)과 외부 대기(4팀 예산 시트) — §9 참고.
> **⚠ repo `Code.gs`는 참고용 사본 — §14 배포본 반영본으로 같이 커밋할 것**(운영자가 Apps Script 편집기엔 배포했으나 repo 사본은 별도 갱신 필요).

## 4. 시트 구조 (몽골 시트 기준, 전 팀 동일 템플릿)

**`수입지출내역서` 탭** = 원본(raw). 앱이 입력 순서대로 맨 아래에 행을 추가한다. 데이터는 **12행부터**.

| 열 | 내용 | 비고 |
|---|---|---|
| A | 일자 | 앱이 **진짜 날짜값**으로 기록 (정렬 위해) |
| B | 계정과목 | `01.항공료`…`12.평가회` / `수입_항공료` 등 |
| C | 지원금/자부담 | |
| D | 내역 | |
| E | 수입(원화) | **시트 수식** (12행에서 복사) |
| F | 지출(원화) | **시트 수식** |
| G | 사용 금액 | 앱 입력 |
| H | 사용 통화 | 원화/달러/현지화1/현지화2 |
| I | 영수증 | O/X |
| J | 비고 | **회계 메모용** (사진 링크 아님) |
| K | 사진 | **사진 드라이브 링크** (서버가 기록) |
| L | id (숨김) | 앱이 동기화 시 기록. 수정·삭제·재시도·중복검사·**머지**의 행 식별 기준. 보기 탭(`수입지출내역서01`)은 A:K만 참조라 안 딸려옴 |
| M | 영수증번호 (숨김) | **[Phase 2 신설]** 서버 단일 발급 `max(M)+1`. **숫자 서식 강제 필수**(`setNumberFormat("0")`) — 날짜 서식 셀이면 `3`을 날짜로 오인해 epoch 오염됨. 보기 탭 A:K 밖이라 숨김 OK |

**환율 셀**: `I3`=달러 기준환율(예 1521), `I6`=현지화1(예 MNT 0.43), `I9`=현지화2(예 BAHT 36).

**E12 / F12 수식** (수입/지출은 `LEFT($B12,2)="수입"`으로 분기):
```
E12 =IF(LEFT($B12,2)="수입",IFERROR(ROUND(IF($H12="원화",$G12,IF($H12="달러",$I$3*$G12,IF($H12="현지화1",$G12*$I$6,IF($H12="현지화2",$G12*$I$9,"")))),0),""),"")
F12 =IF(LEFT($B12,2)="수입","",IFERROR(ROUND(IF($H12="원화",$G12,IF($H12="달러",$I$3*$G12,IF($H12="현지화1",$G12*$I$6,IF($H12="현지화2",$G12*$I$9,"")))),0),""))
```

**`_synced` 숨김 탭**: Phase 1부터 **주 식별자는 L열**(create 시 더 이상 _synced에 쓰지 않음). _synced는 L열이 없는 **구행(이전 배포에서 만들어진 행)의 재시도 폴백** 용으로만 남아 있다.

## 5. 앱 동작 / 데이터 모델

- **계정과목**: 지출 `01.항공료, 02.여행자보험, 03.비자비, 04.준비비, 05.식비, 06.숙박비, 07.교통비, 08.사역비, 09.문화체험, 10.선교헌금, 11.예비비, 12.평가회` / 수입 `수입_항공료, 수입_체제비, 수입_아웃리치, 수입_기타수입`
- **5개 팀**: 몽골, 중국1, 인도네시아1, 말레이시아1, 스리랑카 (기기당 1팀 고정). 현지화1 통화명 팀 고정 `LOCAL1`={몽골:MNT, 중국1:CNY, 인도네시아1:IDR, 말레이시아1:MYR, 스리랑카:LKR} — 앱 표시·시트 환전내역 J3 동일 글자.
- **IndexedDB** `igw_mission` / `entries`. 엔트리 형태:
  `{ id, team, type, account, fund, date, desc, currency, amount, krw, note, receipt, receiptNo, photos:[{b64,size}], photoCount, syncState, photoFail, syncedRow, createdAt }`
- **사진 처리 (중요)**:
  - 촬영 → 캔버스로 **1280px / JPEG 0.7 압축**
  - **압축 직후(메모리 신선할 때) base64 텍스트로 변환해 저장** — 아이폰 IndexedDB가 저장한 blob을 나중에 읽을 때 깨지는 버그를 피하기 위함
  - 변환 안 되는 사진(특이 형식)은 **건너뛰고 "다시 촬영" 안내** (깨진 채 업로드 방지)
- **`syncState` 4값** (Phase 1): `대기`(신규) / `완료` / `수정대기` / `삭제대기`. 동기화 루프가 syncState로 분기 처리.
- **동기화**: 연결되면 한 건씩 POST(text/plain — preflight 회피). 사진은 `[{data,size}]`로 함께 전송. 끊기면 중단 후 다음에 이어서(완료 건은 다시 안 보냄). 앱 열 때 / 온라인 복귀 시 / 수동 버튼으로 실행.
  - 대기 → create (행 추가 + L열 id 기록 + 사진 업로드)
  - 수정대기 → `action:"update"` (A:D · G:J만 덮어쓰고 E·F 수식·K 사진·L id 보존). 응답 `missing:true` = 시트에서 수동 삭제됨 → 앱이 유령 카드 로컬 정리.
  - 삭제대기 → `action:"delete"` (사진 휴지통 → 행 삭제) → 로컬 엔트리 제거. 멱등.
- **수정/삭제 UX**: 내역 항목 탭 → 액션시트(수정/삭제). 삭제는 confirm.
  - 수정폼은 입력폼 재사용, **텍스트·숫자만**(사진 변경은 기존 "다시 올리기"로 교체).
  - **상태 전이**: 완료→수정 ⇒ 수정대기 / 완료→삭제 ⇒ 삭제대기 / 대기→수정 ⇒ 로컬값만(대기 유지) / 대기→삭제 ⇒ 로컬 즉시 제거 / 수정대기→삭제 ⇒ 삭제대기(**delete wins**) / 삭제대기→수정 ⇒ UI에서 비활성.
- **서버 사진 무결성 검사**: 앱이 보낸 `size`와 서버가 디코드한 바이트 길이를 비교. 다르면 깨진 파일을 저장하지 않고 K열에 `⚠ 사진 N 전송 손상(받은크기/보낸크기)` 표시.
- **사진 실패 표시 + 다시 올리기**: 서버 응답에 `photoFail`(실패 장수>0 여부) 포함. 실패면 앱 내역에 `⚠ 사진 실패` 배지 + `사진 다시 올리기` 버튼 표시. 버튼으로 새 사진을 보내면 `_synced`의 `[id, row]`로 원래 행을 찾아 K열만 교체(앱 `retryPhotos` ↔ 서버 `retryPhotos_`·`savePhotos_`).
- **드라이브 저장**: 상위 폴더 안에 팀 폴더 자동 생성. 파일명 `{영수증번호3자리}_{지출일}_{계정과목}.jpg` (여러 장이면 `_1`,`_2`). 예: `003_2026-06-25_01.항공료.jpg`. 기본 공유 `private`(운영자만 열람) — Code.gs `PHOTO_SHARE="view"`로 바꾸면 링크 가진 사람 보기 가능.

### 운영 / 기기 모델 (확정)
- **배포 모델**: 회계별 개인폰에 링크로 PWA 설치. 기기 공유·넘김 없음 → **1인 1폰 1데이터**. 따라서 기기 인수인계용 초기화 기능은 불필요. **앱 내 '로컬 데이터 비우기' 버튼은 의도적으로 두지 않음**(미동기화분 유실 위험 회피).
- **로컬 저장 위치 구분 — 두 곳, 갱신 정책 다름**:
  - 입력 내역(엔트리·사진 base64): **IndexedDB** `igw_mission` / `entries`. **`?v=N` 새로고침이나 sw 버전 올림으로 지워지지 않음**(별개 저장소).
  - 앱 껍데기(html/js/icon/manifest): **서비스워커 캐시**. `sw.js` `CACHE` 버전을 올려야 갱신됨.
- **유령 카드**: 시트에서 운영자가 손으로 지운 행은 그 폰 IndexedDB에 "완료" 상태로 남는다. 다음 수정 동기화 시 서버가 `missing:true`로 응답 → 앱이 그때 로컬 자동 정리. (Phase 2 전엔 능동 폴링 없음 — 회계가 그 항목을 다시 만지지 않으면 유령 카드는 그대로 유지됨)
- **기기 로컬 초기화가 필요할 때**(주로 운영자 테스트 정리, 실사용 중엔 권장 X): PWA 아이콘 삭제 → 사파리에서 사이트 열기 → "홈 화면에 추가"로 새로 설치 (IndexedDB가 새로 시작됨).

## 6. 배포 방법

**터미널(Claude Code)에서 — 프런트엔드**
```
# repo에서 index.html / sw.js 수정 후
git add index.html sw.js
git commit -m "한 토픽 요약"
git push
```
- 프런트 파일을 고치면 **항상 sw.js `CACHE` 버전 +1**. 폰에서 `https://gukja79.github.io/igw-mission/?v=N`로 새로고침해 새 버전 확인.

**브라우저에서 — 백엔드 (Code.gs)**
- repo push로는 **반영 안 됨.** script.google.com 편집기에서 새 코드 붙여넣기 → 저장 → **배포 관리 → 편집(연필) → 버전: 새 버전 → 배포**. **URL 그대로**.
- **⚠ 형식 일치 주의**: index.html이 보내는 데이터 형식을 바꾸면(예: 사진을 `{data,size}`로) **Code.gs도 반드시 재배포**해야 한다. 안 맞으면 사진이 깨진다.
- 드라이브 권한이 새로 필요할 때(사진 기능 최초)는 함수 1회 실행 → 권한 검토에서 Drive 허용(확인되지 않은 앱 경고 시 고급→이동→허용).

## 7. 완료된 것

- 오프라인 입력 + IndexedDB + 사진 촬영/압축 + 내역 목록
- 입력 시 ≈원화 미리보기 + 팀별 환율 설정 화면
- 시트 동기화(행) — 몽골 검증 완료
- 사진 → 드라이브(상위폴더/팀폴더) + K열 링크 — 몽골 검증 완료

## 11. Phase 2 상세 — 앱↔시트 양방향 동기화 (직전 세션 완료)

설계 중 **onEdit 설치형 트리거는 폐기**하고 **pull 시점 보정**으로 대체했다. 이유: 웹 입력은 회계가 메인이지만 가끔 팀리더·회계담당자(서로 다른 구글 계정)도 함. 설치형 트리거는 설치한 계정의 편집에만 발화 → 남의 편집은 누락. 트리거를 각 계정에 깔 수도 없음(스크립트는 운영자 소유). pull 보정은 누가 입력했든 누락 0 + 설치 부담 0.

### 서버 (Code.gs) — 추가/변경 함수
- **`pullAll_(body)`** (`action:"pull"`): 시트 전체를 앱 거울용 JSON 으로 반환. 각 행 `{row,id,receiptNo,date,account,fund,desc,krwIn,krwOut,amount,currency,receipt,note,photo}`. E·F는 시트 수식 **계산값**을 읽음. + **보정(backfill)**: id(L) 비었고 A·B·C·D·G·H 다 찬 '완성행'은 이때 L=id·M=영수증번호 자동 부여(위→아래). `LockService` 직렬화 + 락 안에서 L열 재확인(동시 pull 중복발급 방지). 멱등(이미 채워진 행 재부여 안 함). 응답에 `backfilled:N`.
- **create 변경**: 영수증번호를 서버가 발급. `nextReceiptNo_(sh)`=max(M)+1, `LockService`로 직렬화. M열 기록 시 `setNumberFormat("0")`. 사진 파일명도 서버 발급번호 사용. 앱이 보낸 `receiptNo`는 무시. 응답에 `receiptNo` 포함.
- **헬퍼 신설**: `nextReceiptNo_` / `receiptNoOf_`(M 읽기 안전변환: 빈칸·Date오염→null) / `cellFilled_` / `rowComplete_`(A·B·C·D·G·H) / `makeBackfillId_`(밀리초-랜덤+'w').
- **hotfix**: M열 날짜서식 오염 방어 — write 양쪽 `setNumberFormat("0")`, read는 `receiptNoOf_()`로 Date→null. (이미 오염된 셀은 코드로 못 고침 → 시트에서 서식 일반 + 값 직접 입력으로 수동 정리.)

### 앱 (index.html) — 추가/변경
- **머지**: `pullNow(manual)` (pull→`mergeFromSheet_`→렌더, **시트로 push 안 함**). 변환 헬퍼 `sheetRowToEntry_`/`applySheetValues_`/`sheetDiffers_`. type은 `krwIn>0`이면 수입.
- **머지 규칙 (id 매칭)**:
  - 시트만 있음 → 신규 추가 (`syncState:"완료"`, `syncedRow`, `sheetPhoto`=K링크)
  - 양쪽 + 로컬 `완료` → **시트값으로 갱신**(A). 단 로컬 사진(base64)·`createdAt` 보존.
  - 양쪽 + 미동기화(`대기`/`수정대기`/`삭제대기`) → **로컬 유지**. 시트와 값 다르면 `mirrorFlag:"시트와다름"`(자동 안 덮음).
  - 로컬만 + `완료` → **보존 + `mirrorFlag:"시트에없음"`**(B: 시트에서 삭제된 행). 자동삭제 안 함.
  - 로컬만 + 미동기화 → 보존(곧 push), 표시 없음.
- **엔트리 추가 필드**: `receiptNo`(서버 발급, 폰 localStorage 카운터 `igw_seq_*` 폐기) / `sheetPhoto`(내려받은 행의 K 링크) / `mirrorFlag`.
- **UI**: 동기화 옆 **"내려받기" 버튼**(`pullNow(true)`) · mirror 배지(노랑) · 원화 강조(외화행: 원화=메인 굵게, 외화=같은 크기 굵기없이 흐리게) · **⋯ 액션메뉴**(수정·삭제, 카드 본체 탭 무반응) · **사진 라이트박스**(앱 b64 썸네일 탭→전체화면, 탭 닫기) · 비고는 배지 아닌 **하단 📝 메모 줄**.
- **사진 정책**: 웹은 촬영 불가 → 사진은 앱 입력 행에만 존재(내려받기로 사진 받을 일 거의 없음). 앱 직접올림=썸네일+라이트박스 / 내려받은 시트행(드묾)=`📎 사진 보기` 링크(private라 권한자만) / 사진 없으면 "사진 없음".

3. **깨진 b64 폴백** — 목록 썸네일·라이트박스 `<img>`에 `onerror="imgBroken(this)"`. 디코드 실패 시 깨진 아이콘 대신 `⚠ 사진 깨짐` 박스(`.img-broken`)로 교체. 정상 사진엔 무반응, 원본은 드라이브에 안전하니 표시만의 문제.
4. **업데이트 안내 바** — `updatefound`/`statechange`로 새 SW 설치 감지 → 하단에 "새 버전이 있어요 → 새로고침" 바(`showUpdateBar`, `.updbar`). 탭하면 `location.reload()`. **이 코드가 깔린 v11 이후 배포부터 동작**(설치 당시 도는 SW엔 감지코드 없음). v11→v12 전환에서 첫 실전 검증됨.
5. **버전 배지** — 환율 설정 화면 맨 아래 `.appver`. **하드코딩 아님** — `caches.keys()`로 활성 sw 캐시명에서 `vN`을 읽어 표시(`showAppVersion`). **sw.js `CACHE`만 올리면 배지 자동 반영**(이중 관리 없음). 용도: 운영자 원격 디버깅("설정 맨 아래 숫자 뭐예요?"로 폰 안 보고 버전 확인). 회계용 가치는 거의 없어 작게·흐리게 둠.

### 배포 이력

## 13. Phase 3 상세 — 사진 관리 (이번 세션 완료)

영수증 카드 **⋯ → 사진 관리**에서 **동기화된(완료) 항목**의 사진을 추가·단건삭제. 시트 K열(줄바꿈 다중 링크)이 진실 원본이고, 삭제는 **fileId로 지정**(인덱스 아님)해 꼬임 방지. 앱·시트·드라이브 3곳 일치까지 몽골 검증 완료.

### 서버 (Code.gs) — 추가 함수 + 라우팅
- `doPost` 라우팅 2줄 추가: `addPhotos` / `deletePhoto`.
- **`addPhotos_(body)`** — `{entry:{id,team,photos:[{data,size}]}}`. L열 id로 행 찾기 → 기존 K열 링크 **보존** + 새 사진 드라이브 업로드해 **뒤에 append**. 파일명 번호는 기존 실제 링크 수에서 이어감(`_N`). 크기 무결성 검사 재활용. K열 다시 씀, `photo`(병합본) 반환.
- **`deletePhoto_(body)`** — `{entry:{id,team}, fileId}`(fileId는 top-level). 각 K열 줄에서 fileId를 뽑아 **정확히 일치하는 링크 줄만** 제거(공백/형식 차이에 안전) → 그 파일만 `setTrashed` → K열 다시 씀 → `SpreadsheetApp.flush()`. `removed`(지운 줄 수)·`photo`(남은 K) 반환.
- 기존 함수·다른 라우팅은 무변경. 새 드라이브 권한 동의 없음(기존 `DriveApp` 범위).

### 앱 (index.html) — 추가/변경
- 액션시트에 **사진 관리** 버튼(`act-photos`). 순서 **수정 - 삭제 - 사진 관리 - 취소**(수정·삭제는 영수증 항목 동작이라 인접, 사진 관리는 결이 달라 분리·맨 아래).
- **사진 관리 모달**(`#photo-mgr`): 현재 사진 목록(로컬 b64 있으면 썸네일, 없으면 `📎 사진 N` 링크) + 장당 **삭제** + **사진 추가** 버튼. 온라인 아니면 추가/삭제 비활성 + "연결 필요" 안내(드라이브를 실제로 건드려야 해서 오프라인 큐 불가).
- **`openPhotoMgr`** — 열 때 그 항목 행을 **pull로 읽어 `sheetPhoto`(K링크) 최신화**. 앱에서 직접 입력한 항목은 로컬에 K링크가 없어(pull 안 거침) 이 단계가 없으면 "사진 없음"으로 보임. 살짝 로딩 생김(온라인 시).
- **`pmgrAdd`** — 압축+b64 → `addPhotos` 호출 → `sheetPhoto`·로컬 b64 갱신.
- **`pmgrDelete`** — 삭제 전 링크 순서에서 fileId 위치(idx) 기억 → `deletePhoto` 호출 → `sheetPhoto`를 서버 응답(`photo`) 기준으로 갱신 + **그 idx의 로컬 썸네일 1장만 제거**(앞·중간·뒤 모두 일치). 서버 `removed`가 0이면 "삭제 0" 경고(진단).
- 사진 지정은 K링크에서 뽑은 **fileId**로(`parseSheetLinks_`).

### 배포 이력

커밋: `(Code.gs 참고용 사본)` · `73984d1`(S2 화면+v13) · `c453689`(열 때 pull 최신화+v14) · `5fb7987`(삭제 K열 정확매칭·시트기준·진단+v15) · `(로컬 썸네일 정확위치+v16)` · `50bc44e`(액션시트 순서+v17).

### 알아둘 점 / 검증 중 확인된 것
- **사진 관리는 동기화된(완료) 항목만** — 서버가 L열 id로 행을 찾아야 사진을 건드려서. 미동기화 항목은 "동기화 후 가능" 안내.
- **시크릿(개인정보보호) 탭에선 썸네일 대신 `📎` 링크** — IndexedDB가 비어 로컬 b64가 없어서. 버그 아님, 일반 탭에선 썸네일 정상. 비공개 드라이브 사진은 링크로 `<img>` 직접 표시 불가.
- **카드 "영수증 #N" 표시는 정상** — 시트 M열(서버 발급 번호)과 일치 확인됨(앱 입력분이 1번, 그 위 backfill 손입력행이 2·3·4번이라 번호가 뒤섞여 보이는 것뿐).
- 이전 테스트에서 K열에 남았던 죽은 링크는 자동 청소 안 됨 — 실사용 전 정리(§9-1) 때 손으로 제거(드라이브 파일은 이미 휴지통).
- append 시 원래 1장이던 파일은 접미사 없이 저장돼 있어, 추가분이 `_2`부터 붙어 드라이브 파일명이 약간 안 맞아 보일 수 있음 — **링크는 K열이 진실이라 동작 무관**.

## 14. 환율/환전 자동화 — 시트 + 서버 + 앱 1단계 완료

회계가 **환전 거래만 입력**하면 통화별 가중평균 원화환율이 자동 산출돼 메인 시트 `I3/I6/I9`에 반영되고, 그게 E/F 수식을 타고 현지 지출 원화 환산까지 흐른다. **시트(설계·보조표)·서버(Code.gs fx 라우팅)·앱(환전 입력/목록/동기화) 모두 배포·검증 완료(몽골). 남은 건 2단계(환전 수정 + 사진 관리).**

> 배경: 원화→현지돈 직접 환전이면 환율이 바로 나오지만, **원화→달러→현지화(경유)**나 **여러 날 여러 번 환전** 시 회계가 원화 환산·평균을 힘들어함. 기존엔 시트로 손수 풀었으나(2024 몽골 예결산의 K/L·N~Q·R2 영역) 회계가 다루기 어려워 자동화하기로.

### 설계 결정 (확정)
- **평균 방식 = 금액가중평균**: `(들어간 원화 총합) ÷ (받은 통화 총량)`, **트립 전체 단일값**(날짜별 X). 단순평균 아님 — 환전 금액이 매번 다르면 단순평균은 틀림.
- **경유 환전**: 달러→현지화는 **달러 평균환율을 거쳐** 원화 환산. 보조표가 달러→현지화1→현지화2 순으로 층층이 참조(아래 수식).
- **역방향(현지화→원화, 남은 돈 반납)**: 평균에서 **자동 제외**(받은통화=원화라 보조표가 안 잡음). 규칙상 남은 현지화는 원화로 환전해 반납. 기록만 원하면 환전내역에 넣어도 평균 영향 0.
- **라운드 정책 (중요)**: **환율(M열·I3/6/9)엔 ROUND 걸지 않음**(풀 정밀도 유지). 깎으면 MNT처럼 1 미만 환율에서 환산 오차 누적(예: 0.42372→0.42면 676만 투그릭에 ~2.5만원 차이). **원화 환산 금액(E/F열)만 `ROUND(...,0)` 정수** — 이게 외부 회계프로그램(나눔셈, `nanumsem.kr`)과 원 단위 일치시키는 자리. 기존 E12/F12에 이미 `ROUND(,0)` 적용돼 있음.
- **환율 표시 서식**: 2자리(`0.00`)로 둬도 무방 — **표시 서식은 계산과 무관**(셀은 풀 정밀도 저장, E/F는 풀 값으로 계산). 회계가 4자리 보면 헷갈려해서 2자리 표시 OK. 단 수식에 `ROUND(M2,2)` 넣지 말 것(값 깎임).

### 환전내역 시트 구조 (몽골 시트에 신설, 탭명 `환전내역`)
```
   A      B       C       D        E        F     G     H(숨김) │  J        K         L           M
 1 일자  준통화  준금액  받은통화  받은금액  메모   사진   id     │ 통화    받은총량   들어간원화   평균환율(원/단위)
 2 ── 환전 기록 (A2부터 아래로, 앱이 추가) ──                   │ 달러
 3                                                              │ (현지화1명)
 4                                                              │ (현지화2명/빈칸)
```
- **B·D = 통화** (드롭다운). 저장값 규칙은 "통화명 관리" 참고.
- **보조표 J1:M4 고정.** 기록(A:H)과 열이 안 겹쳐 기록 늘어도 안 밀림.
- **사진(G) = 줄바꿈 다중 링크**, 메인 K열과 동일 파이프라인 재사용 예정.
- **H열 id 숨김** — 동기화·수정·삭제 행 식별용(메인 L열과 동형).

### 보조표 수식 (검증 완료, 그대로 사용)
J2=`달러`, J3=현지화1 통화명(예 `MNT`), J4=현지화2 통화명(쓰면 입력/안 쓰면 빈칸).
```
달러   K2 =SUMIFS(E:E,D:D,J2,B:B,"원화")
       L2 =SUMIFS(C:C,D:D,J2,B:B,"원화")
       M2 =IFERROR(L2/K2,"")
현지화1 K3 =SUMIFS(E:E,D:D,J3,B:B,"원화")+SUMIFS(E:E,D:D,J3,B:B,J2)
       L3 =SUMIFS(C:C,D:D,J3,B:B,"원화")+SUMIFS(C:C,D:D,J3,B:B,J2)*M2
       M3 =IFERROR(L3/K3,"")
현지화2 K4 =IF($J4="","",SUMIFS(E:E,D:D,J4,B:B,"원화")+SUMIFS(E:E,D:D,J4,B:B,J2)+SUMIFS(E:E,D:D,J4,B:B,J3))
       L4 =IF($J4="","",SUMIFS(C:C,D:D,J4,B:B,"원화")+SUMIFS(C:C,D:D,J4,B:B,J2)*M2+SUMIFS(C:C,D:D,J4,B:B,J3)*M3)
       M4 =IF($J4="","",IFERROR(L4/K4,""))
```
**메인 수입지출내역서 연결**: `I3 ='환전내역'!M2` / `I6 ='환전내역'!M3` / `I9 ='환전내역'!M4`.
- 핵심: M3가 M2를, M4가 M2·M3을 끌어 쓰는 **층층이 구조**라 경유 환산 자동.
- **전제**: 달러→현지화 기록이 있으려면 그 전에 원화→달러 기록이 있어야 M2가 잡힘(실무상 달러 먼저 사니 자연 충족). 필요 시 가드 추가 가능.
- I3/6/9가 빈칸이면 E/F 계산 안 됨 = **"환전 정보 먼저 입력" 강제 장치라 폴백 불필요**(의도된 동작).

### 검증 (몽골, 테스트 데이터)
A2:E4에 ①원화 3,000,000→달러 2,000 ②달러 2,000→MNT 6,760,000 ③원화 500,000→MNT 1,500,000, J2=달러·J3=MNT 입력 →
- **M2(달러)=1,500.0** ✓ (3,000,000/2,000)
- **M3(MNT)=0.42372…** ✓ ((500,000+2,000×1,500)/(1,500,000+6,760,000)=3,500,000/8,260,000) — 달러산 MNT(0.4438)+직접산 MNT(0.3333)가 받은 양으로 가중평균된 값. **경유 환산 정확 확인.**
- 메인 I3=1,500 / I6=0.42 / I9=빈칸 반영 확인.
- (J4를 글자 `현지화2`로 넣으면 M4=0으로 뜸 — 실제론 **빈칸**이어야 정상. 테스트 잔재.)

### 통화명 관리 (저장은 식별, 표시는 통화명)
- **원화·달러·현지화1 = 운영자 고정**(팀별, 몽골 현지화1=MNT). 안 바뀌므로 시트 B·D에 **실제 통화명 그대로 저장**(`MNT`).
- **현지화2 = 회계가 1회 입력**(팀마다 경유지 다름, BAHT는 더미였음). 앱 환율설정에서 이름 넣으면 localStorage 고정 🔒 + **수정 버튼(경고)**. → **B안 채택: 시트 전체를 통화명으로 저장**(코드 `현지화2` 안 씀).
- **이름 변경 시(예 바트→밧)**: 앱이 **내부 코드(슬롯)로 추적**해 해당 환전 기록 B·D + 보조표 J + 메인 환율칸 통화명을 **일괄 새 이름으로 동기화**. 찾기는 코드, 쓰기는 이름. (이름 변경은 트립당 0~1회라 부담 미미.)
- 현지화2 이름 비면 앱 `<select>`·시트 보조표에서 자동 off(빈칸).
- 시트 보호 + 드롭다운 병행 가능(둘 별개, 앱 `setValue`는 통과). 앱 `<select>`는 value=식별/표시=이름(rateCfg), 이름 있는 통화만 옵션.

### 구현 상세 — 서버 (Code.gs, 배포·검증 완료)
환전내역 탭은 메인 내역서와 **데이터 2행부터 / id=H열 / 사진=G열 / 영수증번호 없음 / E·F수식 없음**이라 전용 함수군을 둠. I3/6/9·보조표 J:M은 서버가 안 건드림(시트가 계산).
- **상수**: `FX_SHEET_NAME="환전내역"` / `FX_FIRST_ROW=2` / `FX_ID_COL=8`(H) / `FX_PHOTO_COL=7`(G) / `FX_FOLDER="환전"`(사진 {팀}/환전 하위폴더).
- **라우팅 6종**(doPost): `fxCreate`/`fxPull`/`fxUpdate`/`fxDelete`/`fxRenameCurrency`(+ 사진은 fxCreate에 동봉).
- **`fxCreate_`** — A:F + H id + G사진. 번호발급·수식복사 없음(메인보다 단순). id중복 락컷.
- **`fxPull_`** — 환전 목록 + **`rates`**(보조표 J2:M4 → {usd,local1,local2} 각 {name,rate}) 동봉 + 완성행(A·B·C·D·E) H=id 백필.
- **`fxUpdate_`/`fxDelete_`** — A:F 덮어쓰기(G·H 보존) / G사진 휴지통+행삭제(멱등).
- **`fxRenameCurrency_`** — slot local1→J3 / local2→J4. 환전내역 B·D + J라벨 **락으로 원자 갱신**. **빈 oldName=최초 설정(라벨만, B·D 안 건드림)**. 메인 시트는 코드·셀참조라 무관 → 안 건드림.
- **`fxCcyName_`(중요)** — 앱은 통화 **코드**(원화/달러/현지화1/현지화2)를 보내고, 서버가 보조표 J셀 읽어 **B·D엔 이름**(MNT 등)으로 기록. 보조표 SUMIFS가 D열을 J라벨과 매칭하므로 B·D는 반드시 이름이어야 함. 코드 전송이라 **앱 캐시 상태와 무관(오프라인 안전)**.
- 공용 헬퍼 인자화: `findRowById_(sh,id,firstRow,idCol)` · `trashPhotosFromRow_(sh,row,photoCol)` (기본값=메인, FX는 2/8/7 전달). 새 드라이브 권한 없음.

### 구현 상세 — 앱 (index.html v18→v19, 1단계 배포·검증 완료)
- **IndexedDB v1→v2**: `fx` 스토어 신설(기존 `entries` 보존, 추가형). fx 헬퍼 `fxSave/fxGetAll/fxDel`.
- **환율 직접입력 화면 제거** → **환전 화면(`view-fx`)**: 상단 현재환율 카드(읽기전용, `rates`) + 환전 입력폼(일자/준통화·금액/받은통화·금액/메모/사진) + 환전 목록 + 삭제. `환율` 버튼이 이 화면 염.
- **통화명 = 설정값(오프라인 안정) / 환율값 = 서버**: 원화·달러 고정, 현지화1=`LOCAL1[팀]`, 현지화2=`local2Name`(localStorage `igw_local2_*`). `curLabel_`가 코드→이름. 환율값은 `fxPull rates`로 `rateCfg[*].rate` 채움(라벨은 설정값 유지).
- **현지화2(경유통화) 설정**: 환율카드 `＋ 경유 통화 설정`→prompt→`fxRenameCurrency`(oldName 빈값=최초). 설정되면 드롭다운 등장🔒·이름수정 가능. **연결 필요**(시트 J4 써야 해서).
- **동기화**: `fxSyncNow`(대기→fxCreate / 삭제대기→fxDelete) · `fxRefresh`(fxPull→rates 반영+목록 머지). 저장 직후·온라인복귀·화면열때·startMain 백그라운드 실행. 머지는 경량(id매칭: 신규추가/완료갱신/미동기화보존/시트없음 mirror).
- **금액 천단위 콤마**(v19): 사용금액·환전 준/받은금액 3칸 `type=text`+라이브 포맷(`fmtAmountInput`), 저장·계산은 `parseAmt`로 숫자화. 시트는 콤마 서식 별도라 충돌 없음.
- 원장 ≈원화 미리보기 문구: `(저장된 환율 기준, 연결 시 확정)` / 환율없음 `연결되면 자동 계산`.
