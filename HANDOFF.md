# igw-mission 인수인계 문서

선교 해외 아웃리치 **오프라인 회계 PWA**. 각 팀 회계가 휴대폰에서 오프라인으로 지출/수입을 입력하면, 연결될 때 팀별 구글시트로 동기화되고 영수증 사진은 구글 드라이브에 저장된다.

> 최종 업데이트 기준 상태: **Phase 2(앱↔시트 양방향 동기화)까지 완료·검증.** 몽골팀 운영 중(나머지 6팀 미연결). 앱 캐시 버전 `v10`, 최종 커밋 `01ad1bf`(HANDOFF는 그 이후).

## 2. 스택 / 아키텍처

- **프런트엔드**: GitHub Pages (정적 HTML/JS, 서비스워커로 오프라인 동작)

## 4. 시트 구조 (몽골 시트 기준, 전 팀 동일 템플릿)

**`수입지출내역서` 탭** = 원본(raw). 앱이 입력 순서대로 맨 아래에 행을 추가한다. 데이터는 **12행부터**.

| 열 | 내용 | 비고 |
|---|---|---|
| A | 일자 | 앱이 **진짜 날짜값**으로 기록 (정렬 위해) |
| B | 계정과목 | `01.항공료`…`12.평가회` / `수입_항공료` 등 |
| C | 지원금/자부담 | |
| D | 내역 | |
| E | 수입(원화) | **시트 수식** (12행에서 복사) |
| F | 지출(원화) | **시트 수식** |
| G | 사용 금액 | 앱 입력 |
| H | 사용 통화 | 원화/달러/현지화1/현지화2 |
| I | 영수증 | O/X |
| J | 비고 | **회계 메모용** (사진 링크 아님) |
| K | 사진 | **사진 드라이브 링크** (서버가 기록) |
| L | id (숨김) | 앱이 동기화 시 기록. 수정·삭제·재시도·중복검사·**머지**의 행 식별 기준. 보기 탭(`수입지출내역서01`)은 A:K만 참조라 안 딸려옴 |
| M | 영수증번호 (숨김) | **[Phase 2 신설]** 서버 단일 발급 `max(M)+1`. **숫자 서식 강제 필수**(`setNumberFormat("0")`) — 날짜 서식 셀이면 `3`을 날짜로 오인해 epoch 오염됨. 보기 탭 A:K 밖이라 숨김 OK |

**환율 셀**: `I3`=달러 기준환율(예 1521), `I6`=현지화1(예 MNT 0.43), `I9`=현지화2(예 BAHT 36).

**E12 / F12 수식** (수입/지출은 `LEFT($B12,2)="수입"`으로 분기):
```
E12 =IF(LEFT($B12,2)="수입",IFERROR(ROUND(IF($H12="원화",$G12,IF($H12="달러",$I$3*$G12,IF($H12="현지화1",$G12*$I$6,IF($H12="현지화2",$G12*$I$9,"")))),0),""),"")
F12 =IF(LEFT($B12,2)="수입","",IFERROR(ROUND(IF($H12="원화",$G12,IF($H12="달러",$I$3*$G12,IF($H12="현지화1",$G12*$I$6,IF($H12="현지화2",$G12*$I$9,"")))),0),""))
```

**`_synced` 숨김 탭**: Phase 1부터 **주 식별자는 L열**(create 시 더 이상 _synced에 쓰지 않음). _synced는 L열이 없는 **구행(이전 배포에서 만들어진 행)의 재시도 폴백** 용으로만 남아 있다.

## 5. 앱 동작 / 데이터 모델

- **계정과목**: 지출 `01.항공료, 02.여행자보험, 03.비자비, 04.준비비, 05.식비, 06.숙박비, 07.교통비, 08.사역비, 09.문화체험, 10.선교헌금, 11.예비비, 12.평가회` / 수입 `수입_항공료, 수입_체제비, 수입_아웃리치, 수입_기타수입`
- **7개 팀**: 라오스1, 몽골, 중국1, 인도네시아1, 말레이시아1, 캄보디아1, 스리랑카 (기기당 1팀 고정)
- **IndexedDB** `igw_mission` / `entries`. 엔트리 형태:
  `{ id, team, type, account, fund, date, desc, currency, amount, krw, note, receipt, receiptNo, photos:[{b64,size}], photoCount, syncState, photoFail, syncedRow, createdAt }`
- **사진 처리 (중요)**:
  - 촬영 → 캔버스로 **1280px / JPEG 0.7 압축**
  - **압축 직후(메모리 신선할 때) base64 텍스트로 변환해 저장** — 아이폰 IndexedDB가 저장한 blob을 나중에 읽을 때 깨지는 버그를 피하기 위함
  - 변환 안 되는 사진(특이 형식)은 **건너뛰고 "다시 촬영" 안내** (깨진 채 업로드 방지)
- **`syncState` 4값** (Phase 1): `대기`(신규) / `완료` / `수정대기` / `삭제대기`. 동기화 루프가 syncState로 분기 처리.
- **동기화**: 연결되면 한 건씩 POST(text/plain — preflight 회피). 사진은 `[{data,size}]`로 함께 전송. 끊기면 중단 후 다음에 이어서(완료 건은 다시 안 보냄). 앱 열 때 / 온라인 복귀 시 / 수동 버튼으로 실행.
  - 대기 → create (행 추가 + L열 id 기록 + 사진 업로드)
  - 수정대기 → `action:"update"` (A:D · G:J만 덮어쓰고 E·F 수식·K 사진·L id 보존). 응답 `missing:true` = 시트에서 수동 삭제됨 → 앱이 유령 카드 로컬 정리.
  - 삭제대기 → `action:"delete"` (사진 휴지통 → 행 삭제) → 로컬 엔트리 제거. 멱등.
- **수정/삭제 UX**: 내역 항목 탭 → 액션시트(수정/삭제). 삭제는 confirm.
  - 수정폼은 입력폼 재사용, **텍스트·숫자만**(사진 변경은 기존 "다시 올리기"로 교체).
  - **상태 전이**: 완료→수정 ⇒ 수정대기 / 완료→삭제 ⇒ 삭제대기 / 대기→수정 ⇒ 로컬값만(대기 유지) / 대기→삭제 ⇒ 로컬 즉시 제거 / 수정대기→삭제 ⇒ 삭제대기(**delete wins**) / 삭제대기→수정 ⇒ UI에서 비활성.
- **서버 사진 무결성 검사**: 앱이 보낸 `size`와 서버가 디코드한 바이트 길이를 비교. 다르면 깨진 파일을 저장하지 않고 K열에 `⚠ 사진 N 전송 손상(받은크기/보낸크기)` 표시.
- **사진 실패 표시 + 다시 올리기**: 서버 응답에 `photoFail`(실패 장수>0 여부) 포함. 실패면 앱 내역에 `⚠ 사진 실패` 배지 + `사진 다시 올리기` 버튼 표시. 버튼으로 새 사진을 보내면 `_synced`의 `[id, row]`로 원래 행을 찾아 K열만 교체(앱 `retryPhotos` ↔ 서버 `retryPhotos_`·`savePhotos_`).
- **드라이브 저장**: 상위 폴더 안에 팀 폴더 자동 생성. 파일명 `{영수증번호3자리}_{지출일}_{계정과목}.jpg` (여러 장이면 `_1`,`_2`). 예: `003_2026-06-25_01.항공료.jpg`. 기본 공유 `private`(운영자만 열람) — Code.gs `PHOTO_SHARE="view"`로 바꾸면 링크 가진 사람 보기 가능.

### 운영 / 기기 모델 (확정)
- **배포 모델**: 회계별 개인폰에 링크로 PWA 설치. 기기 공유·넘김 없음 → **1인 1폰 1데이터**. 따라서 기기 인수인계용 초기화 기능은 불필요. **앱 내 '로컬 데이터 비우기' 버튼은 의도적으로 두지 않음**(미동기화분 유실 위험 회피).
- **로컬 저장 위치 구분 — 두 곳, 갱신 정책 다름**:
  - 입력 내역(엔트리·사진 base64): **IndexedDB** `igw_mission` / `entries`. **`?v=N` 새로고침이나 sw 버전 올림으로 지워지지 않음**(별개 저장소).
  - 앱 껍데기(html/js/icon/manifest): **서비스워커 캐시**. `sw.js` `CACHE` 버전을 올려야 갱신됨.
- **유령 카드**: 시트에서 운영자가 손으로 지운 행은 그 폰 IndexedDB에 "완료" 상태로 남는다. 다음 수정 동기화 시 서버가 `missing:true`로 응답 → 앱이 그때 로컬 자동 정리. (Phase 2 전엔 능동 폴링 없음 — 회계가 그 항목을 다시 만지지 않으면 유령 카드는 그대로 유지됨)
- **기기 로컬 초기화가 필요할 때**(주로 운영자 테스트 정리, 실사용 중엔 권장 X): PWA 아이콘 삭제 → 사파리에서 사이트 열기 → "홈 화면에 추가"로 새로 설치 (IndexedDB가 새로 시작됨).

## 6. 배포 방법

**터미널(Claude Code)에서 — 프런트엔드**
```
# repo에서 index.html / sw.js 수정 후
git add index.html sw.js
git commit -m "한 토픽 요약"
git push
```
- 프런트 파일을 고치면 **항상 sw.js `CACHE` 버전 +1**. 폰에서 `https://gukja79.github.io/igw-mission/?v=N`로 새로고침해 새 버전 확인.

**브라우저에서 — 백엔드 (Code.gs)**
- repo push로는 **반영 안 됨.** script.google.com 편집기에서 새 코드 붙여넣기 → 저장 → **배포 관리 → 편집(연필) → 버전: 새 버전 → 배포**. **URL 그대로**.
- **⚠ 형식 일치 주의**: index.html이 보내는 데이터 형식을 바꾸면(예: 사진을 `{data,size}`로) **Code.gs도 반드시 재배포**해야 한다. 안 맞으면 사진이 깨진다.
- 드라이브 권한이 새로 필요할 때(사진 기능 최초)는 함수 1회 실행 → 권한 검토에서 Drive 허용(확인되지 않은 앱 경고 시 고급→이동→허용).

## 7. 완료된 것

- 오프라인 입력 + IndexedDB + 사진 촬영/압축 + 내역 목록
- 입력 시 ≈원화 미리보기 + 팀별 환율 설정 화면
- 시트 동기화(행) — 몽골 검증 완료
- 사진 → 드라이브(상위폴더/팀폴더) + K열 링크 — 몽골 검증 완료
- 안정화: 캡처 시 base64 변환(아이폰 대응), 변환 불가 사진 건너뛰기, 서버 사진 크기 무결성 검사
- 사진 실패 표시(`⚠ 사진 실패` 배지) + 다시 올리기(행 유지, K열만 교체) — 몽골 검증 완료 (앱 v6 / Code.gs 재배포)
- 정렬 보기 탭 `수입지출내역서01` — 몽골 완료. 원본을 계정과목 목록(`예결산안!B33:B48`) 순서 → 날짜순으로 자동 정렬. A12 수식: `=IFERROR(SORT(FILTER(수입지출내역서!A12:K, 수입지출내역서!B12:B<>""), ARRAYFORMULA(MATCH(FILTER(수입지출내역서!B12:B, 수입지출내역서!B12:B<>""), 예결산안!$B$33:$B$48, 0)), TRUE, 1, TRUE), "")` — 읽기 전용(12행 이하 직접입력 금지), 정렬 순서는 B33:B48 목록으로 제어.
- 앱 수정/삭제 **Phase 1** — L열 id 식별 + `update`/`delete` 분기 + 오프라인 큐 4상태(`대기`/`완료`/`수정대기`/`삭제대기`). 몽골 검증 완료 (신규·수정·삭제, 단일·다중 사진 삭제 시 휴지통 이동). 서버 `findRowById_`·`updateEntry_`·`deleteEntry_`·`trashPhotosFromRow_` / 앱 액션시트·수정폼·`postAction`. (앱 v7 / Code.gs 재배포)
- **Phase 2 — 앱↔시트 양방향 동기화** (이번 세션 완료, 아래 §11 상세). pull(읽기+보정) · 영수증번호 서버발급+락 · pull 보정 backfill · M열 hotfix / 앱 머지코어·내려받기 버튼·mirror 배지·원화강조·⋯ 액션메뉴·사진 라이트박스·비고 메모. 몽골 검증 완료 (앱 v10 / Code.gs 재배포).

## 8. 현재 상태 / 실사용 전 할 일

## 11. Phase 2 상세 — 앱↔시트 양방향 동기화 (이번 세션 완료)

설계 중 **onEdit 설치형 트리거는 폐기**하고 **pull 시점 보정**으로 대체했다. 이유: 웹 입력은 회계가 메인이지만 가끔 팀리더·회계담당자(서로 다른 구글 계정)도 함. 설치형 트리거는 설치한 계정의 편집에만 발화 → 남의 편집은 누락. 트리거를 각 계정에 깔 수도 없음(스크립트는 운영자 소유). pull 보정은 누가 입력했든 누락 0 + 설치 부담 0.

### 서버 (Code.gs) — 추가/변경 함수
- **`pullAll_(body)`** (`action:"pull"`): 시트 전체를 앱 거울용 JSON 으로 반환. 각 행 `{row,id,receiptNo,date,account,fund,desc,krwIn,krwOut,amount,currency,receipt,note,photo}`. E·F는 시트 수식 **계산값**을 읽음. + **보정(backfill)**: id(L) 비었고 A·B·C·D·G·H 다 찬 '완성행'은 이때 L=id·M=영수증번호 자동 부여(위→아래). `LockService` 직렬화 + 락 안에서 L열 재확인(동시 pull 중복발급 방지). 멱등(이미 채워진 행 재부여 안 함). 응답에 `backfilled:N`.
- **create 변경**: 영수증번호를 서버가 발급. `nextReceiptNo_(sh)`=max(M)+1, `LockService`로 직렬화. M열 기록 시 `setNumberFormat("0")`. 사진 파일명도 서버 발급번호 사용. 앱이 보낸 `receiptNo`는 무시. 응답에 `receiptNo` 포함.
- **헬퍼 신설**: `nextReceiptNo_` / `receiptNoOf_`(M 읽기 안전변환: 빈칸·Date오염→null) / `cellFilled_` / `rowComplete_`(A·B·C·D·G·H) / `makeBackfillId_`(밀리초-랜덤+'w').
- **hotfix**: M열 날짜서식 오염 방어 — write 양쪽 `setNumberFormat("0")`, read는 `receiptNoOf_()`로 Date→null. (이미 오염된 셀은 코드로 못 고침 → 시트에서 서식 일반 + 값 직접 입력으로 수동 정리.)

### 앱 (index.html) — 추가/변경
- **머지**: `pullNow(manual)` (pull→`mergeFromSheet_`→렌더, **시트로 push 안 함**). 변환 헬퍼 `sheetRowToEntry_`/`applySheetValues_`/`sheetDiffers_`. type은 `krwIn>0`이면 수입.
- **머지 규칙 (id 매칭)**:
  - 시트만 있음 → 신규 추가 (`syncState:"완료"`, `syncedRow`, `sheetPhoto`=K링크)
  - 양쪽 + 로컬 `완료` → **시트값으로 갱신**(A). 단 로컬 사진(base64)·`createdAt` 보존.
  - 양쪽 + 미동기화(`대기`/`수정대기`/`삭제대기`) → **로컬 유지**. 시트와 값 다르면 `mirrorFlag:"시트와다름"`(자동 안 덮음).
  - 로컬만 + `완료` → **보존 + `mirrorFlag:"시트에없음"`**(B: 시트에서 삭제된 행). 자동삭제 안 함.
  - 로컬만 + 미동기화 → 보존(곧 push), 표시 없음.
- **엔트리 추가 필드**: `receiptNo`(서버 발급, 폰 localStorage 카운터 `igw_seq_*` 폐기) / `sheetPhoto`(내려받은 행의 K 링크) / `mirrorFlag`.
- **UI**: 동기화 옆 **"내려받기" 버튼**(`pullNow(true)`) · mirror 배지(노랑) · 원화 강조(외화행: 원화=메인 굵게, 외화=같은 크기 굵기없이 흐리게) · **⋯ 액션메뉴**(수정·삭제, 카드 본체 탭 무반응) · **사진 라이트박스**(앱 b64 썸네일 탭→전체화면, 탭 닫기) · 비고는 배지 아닌 **하단 📝 메모 줄**.
- **사진 정책**: 웹은 촬영 불가 → 사진은 앱 입력 행에만 존재(내려받기로 사진 받을 일 거의 없음). 앱 직접올림=썸네일+라이트박스 / 내려받은 시트행(드묾)=`📎 사진 보기` 링크(private라 권한자만) / 사진 없으면 "사진 없음".
